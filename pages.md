5.1. Страница входа и регистрации
-	Кнопки «Вход» и «Регистрация».
-	Форма входа: поле «Логин», поле «Пароль», кнопка «Войти».
-	Форма регистрации: поле «Логин», поле «Пароль», поле «Подтверждение пароля», кнопка «Зарегистрироваться».
5.2. Страница профиля пользователя
-	Аватар пользователя
-	Логин
-	Дата регистрации.
-	Кнопка «Удалить аккаунт» (с подтверждением).
-	Вкладки:
-	«Мои инциденты» (для пользователя): таблица созданных пользователем инцидентов (название, статус, дата создания), клик по строке ведёт на страницу инцидента.
-	«Новые инциденты» (для модератора): таблица инцидентов со статусом «новый», не привязанных ни к одному микросервису. При нажатии на строку инцидента происходит переход на страницу деталей инцидента, где модератор может выполнить распределение.
-	«Мои микросервисы» (для разработчика и администратора): таблица микросервисов: название, статус (активен/архивирован), иконка избранного. Клик по строчке переводит на страницу конкретного микросервиса
-	Для руководителя проекта кнопка «Создать микросервис»
-	Для руководителя на каждой строчке таблицы кнопки «Удалить» с запросом подтверждения и «Изменить».
-	«Уведомления»: список уведомлений, отсортированный по дате (свежие сверху). Непрочитанные выделены жирным шрифтом.
-	Для каждого уведомления: текст уведомления, временная метка, ссылка на связанный инцидент.
-	Кнопка «Отметить все прочитанными».
-	Кнопка «Выйти»
5.3. Страница инцидента
-	Название инцидента, статус (с возможностью смены для разработчика/модератора), автор, дата создания
-	Описание инцидента
-	Шаги для воспроизведения
-	Приоритет, Дедлайн. Для модератора кнопка «Изменить». Не доступно пользователю.
-	Связанные микросервисы (перечень). Не доступно пользователю.
-	Назначенные разработчики: список разработчиков, назначенных модератором. Не доступно пользователю.
-	Кнопка «Распределить» (только для модератора, если статус «новый»): открывает модальное окно с выбором микросервисов и назначением разработчиков; кнопка «Сохранить»
-	Кнопки верификации для автора: «Принять исправление» и «Вернуть в работу» — отображаются при статусе «Исправлен»
-	Вкладки:
-	«Комментарии» (активна по умолчанию): список комментариев; поле ввода текста, кнопка «Отправить»
-	«История изменений»: таблица с полями дата, пользователь, изменённое поле, старое значение, новое значение
5.4. Страница создания инцидента
-	Поле «Название» (обязательное)
-	Поле «Описание»
-	Поле «Шаги для воспроизведения»
-	Кнопка «Создать»
-	Кнопка «Отмена»
5.5. Страница микросервиса
-	Название, описание, дата создания, статус
-	Кнопка «Изменить» (для руководителя).
-	Вкладки:
-	«Команда»: список участников с указанием роли (модератор/разработчик) и даты добавления. Для руководителя кнопка «Добавить участника» (открывает форму выбора пользователя), для каждого участника кнопка «Удалить».
-	«Инциденты»: таблица инцидентов, привязанных к данному микросервису. При нажатии на строку переход на страницу инцидента.
5.6. Страница пользователей
Доступна только администратору
-	Таблица всех пользователей: логин, роль, дата регистрации.
-	Фильтр по роли (выпадающий список).
-	Кнопка «Создать пользователя»: форма с полями Логин, Роль (модератор/разработчик), кнопка «Создать» (пароль генерируется автоматически).
-	Для каждого пользователя кнопка «Удалить».
